# Исправление DNS записей для daribri.ru

## Проблема

DNS записи для `daribri.ru` указывают на неправильный IP адрес. Нужно обновить их на `147.45.100.252`.

## Решение

### Шаг 1: Определите, где управляется домен

Домен может управляться:
1. **У регистратора домена** (где вы покупали домен)
2. **У DNS провайдера** (если используете внешний DNS, например Cloudflare, Яндекс DNS и т.д.)

### Шаг 2: Найдите панель управления DNS

**Если домен у регистратора:**
- Зайдите в личный кабинет регистратора домена
- Найдите раздел "DNS", "Управление DNS", "DNS записи" или "Зона DNS"

**Популярные регистраторы:**
- **Reg.ru** → Раздел "DNS-серверы и зона"
- **Timeweb** → Раздел "DNS"
- **REGISTRAR.RU** → Раздел "DNS-зона"
- **Яндекс.Коннект** → Раздел "DNS"
- **GoDaddy** → DNS Management
- **Namecheap** → Advanced DNS

### Шаг 3: Обновите A записи

Найдите существующие A записи для:
- `daribri.ru` (или `@`)
- `www.daribri.ru` (или `www`)

**Измените их на:**
- **Тип:** `A`
- **Имя:** `@` (или `daribri.ru`, или оставьте пустым - зависит от регистратора)
- **Значение:** `147.45.100.252`
- **TTL:** `3600` (или автоматически)

- **Тип:** `A`
- **Имя:** `www`
- **Значение:** `147.45.100.252`
- **TTL:** `3600` (или автоматически)

### Шаг 4: Удалите неправильные записи

Если есть старые A записи с другими IP адресами, **удалите их**.

### Шаг 5: Сохраните изменения

После обновления DNS записей:
1. **Сохраните изменения** в панели управления
2. **Подождите 5-30 минут** для распространения DNS (иногда до 24 часов)

### Шаг 6: Проверьте обновление DNS

```bash
# Проверьте DNS записи
dig daribri.ru A +short
dig www.daribri.ru A +short

# Должно вернуть: 147.45.100.252

# Или через nslookup
nslookup daribri.ru
nslookup www.daribri.ru
```

**Проверьте через онлайн сервисы:**
- https://dnschecker.org/#A/daribri.ru
- https://www.whatsmydns.net/#A/daribri.ru

### Шаг 7: После обновления DNS

После того, как DNS обновится (проверьте через `dig daribri.ru A +short`):

```bash
# 1. Проверьте доступность домена
curl -I http://daribri.ru
# Должен вернуть HTTP ответ

# 2. Если всё ОК, получите SSL сертификат
sudo certbot --nginx -d daribri.ru -d www.daribri.ru
```

## Проверка NS серверов

Если DNS не обновляется, проверьте NS серверы:

```bash
# Проверьте NS серверы
dig NS daribri.ru +short
nslookup -type=NS daribri.ru
```

**Убедитесь, что NS серверы правильные:**
- Если используете DNS регистратора - должны быть NS серверы регистратора
- Если используете внешний DNS (Cloudflare) - должны быть NS серверы Cloudflare

**Если NS серверы неправильные:**
1. Зайдите в панель управления доменом у регистратора
2. Найдите раздел "NS серверы" или "DNS серверы"
3. Обновите NS серверы на правильные

## Ускорение распространения DNS

После обновления DNS записей:

1. **Очистите DNS кеш на вашем компьютере:**
   ```bash
   # Windows
   ipconfig /flushdns
   
   # Linux/Mac
   sudo systemd-resolve --flush-caches
   # или
   sudo dscacheutil -flushcache
   ```

2. **Используйте публичные DNS серверы для проверки:**
   ```bash
   # Google DNS
   dig @8.8.8.8 daribri.ru A +short
   
   # Cloudflare DNS
   dig @1.1.1.1 daribri.ru A +short
   ```
